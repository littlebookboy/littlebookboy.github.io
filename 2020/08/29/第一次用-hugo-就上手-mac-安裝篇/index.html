第一次用 Hugo 就上手 - Mac 安裝篇

2020-08-29 11:02:39 UTC+8 分類 dev

紀錄下我使用 Hugo 架設個人部落格的過程，本篇主要講安裝，與基本使用。

另外，因為我個人的需要，在佈景 (Theme) 部分設置上，有調整一些細部的 config 設置，因此想說相關紀錄另寫一篇文章，會比較詳細 (欠著待補的文章統一列在 坑 中)。

Install 安裝

本篇以 Mac 為主，我的作業系統是 macOS Catalina，若需其他系統、環境的安裝方式，請參考 Hugo Getting Started - Installing 。

安裝步驟如下:

  • 安裝 Hugo (CLI)

    我選擇透過 Homebrew 安裝 Hugo。平常我習慣盡量統一用同一種方式、途徑安裝新東西，除非有其他考量；安裝習慣一致後，日後在重建工作環境，或是查找原本是如何安裝某指令時，不會忘記原本我曾經是怎麼安裝的。

    brew install hugo

  • 安裝成功後，執行以下指令，測試一下 hugo 指令是否可正常運作:

    hugo version

# output: 
# Hugo Static Site Generator v0.74.3/extended darwin/amd64 BuildDate: unknown

  • 建立新站點 (Site)

    # hugo new site your-site-name
# for example
hugo new site my-hugo-blog

  • 選擇佈景 (Themes)

    你可以到 Hugo Themes 選一個看起來順眼的佈景；我習慣是要先選到一個比較符合我想要的感覺的佈景，才會決定使用這個工具，所以在還沒開始做這些動作時，光是佈景就挑了好幾天，為節省正在觀看文章的你一點時間，在此提供你幾個方向參考參考:

    1. 整體樣式: 你曾經看過，或比較喜歡的部落格的樣式，你想要用跟他一樣，或是雷同的
    2. 風格定位: 例如你是要專門撰寫技術類文章，或是 landinig page，可以透過 Hugo Theme 右側列出的 tag 去查找
    3. 內外佈局: 我會看文章 read more 展開之後的感覺，與首頁文章列表的佈局，以及你是否需要有側邊欄 (Menu)，或是導航列 (Navbar) 等等
    4. 選單圖案: 佈景提供的 Menu 連結搭配的 icon 樣式、排列，與感覺
    5. markdown style: 通常佈景都會有數篇 demo 文章供你參考，其中通常都會有 markdown sample post 可參考
    6. code block style: 佈景提供的 code block 樣式是不是你喜歡的樣子，是否提供行號顯示，或是該樣式是否方便你透過 config 做替換
    7. plugin integration: 例如支援數學公式，流程圖繪製，評論系統 comment system 的整合度、嵌入影片語法等 (不需要自行另外集成)

    其他還有一些細節，在你備選的幾個佈景輪流試用中，根據每種佈景作者提供的參數，去切換參數設置看看，你才知道哪些是可隨你意思做動的部分，而哪些是你以後寫文章 (或是你喜歡的部落格也有使用相同效果) 會需要常用到的。

    我是從「佈局」與「整體感覺」先挑出幾個候選，然後每種都設定試試看；在還沒開始寫文章時，我就先用了 jekyll ，但因為選不到我想要的佈景，且看了幾套後覺得 go 很潮 Hugo 標榜速度超快，所以跳來 Hugo 選佈景，試用了 Even 與 Tranquilpeak 之後，才決定用後者。

  • 安裝佈景

    以我自己使用的佈景為例，我是沒有透過 git submodule 的方式安裝，而是直接 clone 到本地，將 clone 下來的專案放到 hugo site 資料夾 themes 底下，這與透過 submodule 抓下來其實是同樣意思

    # 在專案資料夾底下
git clone https://github.com/kakawait/hugo-tranquilpeak-theme.git themes/tranquilpeak

# themes 資料夾是用來專門放置佈景的
# 特地將 hugo-tranquilpeak-theme 重命名為 tranquilpeak，在設置 config 時指定佈景名稱比較方便

    通常每個佈景都會有自己的配置方式，你可以先讀一下佈景的 README.md 後，在跟著佈景的安裝方式走一遍，也可看看佈景有哪些東西供你做調整配置 (config.toml)。

Usage 基本使用

基本上，hugo 指令搞定之後，site 建立完成，佈景設置好，就剩下「建立文章」、「發布上線」這兩個重點步驟。

  • 新增文章

    稍微說明一下，post/xxx.md 這個結構是: 資料夾/檔案名稱+markdown 副檔名，指令執行之後，檔案會長在 /content 底下，例如 /content/post/filename.md

    hugo new post/my-first-post.md

# 官方使用 posts
# 但我這邊都改用 post，似乎會與 Hugo 版本新舊而有差異，但我沒有仔細追查，若有人知道差異歡迎留言跟我分享一下。

  • 本地執行 hugo server 看看結果

    當你尚未把文章丟上線 (Publish) 時，通常會在本地 (Local) 寫文章、改來改去，直到吿一個段落才會發布，在此之前，你可以透過執行以下指令，直接在 localhost:1313 (port 可自行指定，預設是 1313) 查看，文章發佈後會是什麼樣子；

    hugo 目前版本都已有 live reload 模式，我們不用特別下指令 (舊版貌似要用 watch 還是啥)，也就是 server running 期間，修改配置、文章，都會馬上渲染生效。

    hugo server -D

# or `hugo server` running without `draft: true` post
hugo server

  • 本地打包 Hugo

    本地打包就是把你當前的 Hugo 專案，包成一整個可被訪問的靜態資源，產出到資料夾 public，日後你要把網站整個發布到線上時，就是把 public 整包丟到你的 web server 上，不用在另行編譯；你也可以透過指定 publishdir 來變更產出的資料夾名稱。

    hugo

# output (like):
#                     | ZH-TW
#  -------------------+--------
#    Pages            |     2
#    Paginator pages  |     0
#    Non-page files   |     0
#    Static files     |     9
#    Processed images |     0
#    Aliases          |     2
#    Sitemaps         |     1
#    Cleaned          |     0
#  
#  Total in 72 ms

  • 發佈到線上 Web Server

    最後要放到線上，公開你的網站，設置個人網址這部分，會另外在寫一篇文章說明: Hugo 發佈到 GitHub Page 。

Post 文章相關設置

Post Status 文章狀態

以下列出官方定義文章的幾種滿足條件後可能的狀態，我目前只有用到 草稿 的部分:

  • 發布或未發布: Content with a future publishdate value
  • 草稿或已完成: Content with draft: true or false status
  • 有效或已過期: Content with a past expirydate value

而我自己在寫文時，我把文章區分為 page、post，以及 draft:

  • draft 草稿文章 - 尚未完成的文章，本地編譯時需帶參數 -D 才可以看到此類型文章
  • post 已完成文章 - 草稿完成時，將文章改為已完成 (其實就是變更 Front Matter 設定 draft 狀態 true -> false)
  • page 單頁面 - 例如關於我 (about-me)，履歷 (CV)，landing page 等等，這類頁面通常不掛分類 (category) 與標籤 (tag)

Post Archetypes 文章模板

根據上述區分方式，準備各自的模板，模板文件會放在 /archetypes 中 (結構部分可參考 Hugo 資料結構)，以下是我的 post 模板範例: 

---
title: "{{ replace .Name "-" " " | title }}"
author: littlebookboy
type: post
date: {{ .Date }}
draft: true
categories: ["dev"]
tags: ["general"]
keywords: ["post"]
---
